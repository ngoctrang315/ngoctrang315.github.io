Bật CloudWatch Metrics cho Athena

Trước tiên, chúng ta sẽ bật CloudWatch Metrics cho Athena Workgroup chính, giúp theo dõi các số liệu khi chạy truy vấn trong workshop này.

Các bước thực hiện:

  1. Từ menu AWS Services, tìm kiếm Athena và mở Athena Console.
  2. Trong Athena Console, nhấp vào biểu tượng ☰ (ba dấu gạch ngang) ở góc trên bên trái.
  3. Chọn Workgroups và nhấp vào primary workgroup.
  4. Nhấn vào nút Edit.
  5. Cuộn xuống phần Settings, bật tùy chọn Publish query metrics to AWS CloudWatch, sau đó nhấn Save changes.

Cấu hình Vị trí Lưu Kết Quả Athena

  1. Trong Athena Console, nhấp vào biểu tượng ☰, chọn Query Editor.
  2. Nhấn vào Settings, sau đó chọn Manage.
  3. Trong phần Manage Settings, nhấp vào Browse S3.
  4. Tìm bucket có tên: athena-workshop-<account id> (ví dụ: athena-bucket-12345678910). Chọn bucket bằng cách nhấn vào radio button, sau đó nhấn Choose.
  5. Nhấn Save để lưu vị trí trên S3.

Tạo Bảng và Chạy Truy Vấn

Bây giờ chúng ta sẽ tạo một số bảng mới và chạy truy vấn trên chúng.

  1. Nhấp vào tab Editor để mở Athena Query Editor.
  2. Đảm bảo bạn đang ở default database trước khi chạy truy vấn. Kiểm tra trong dropdown Database.
  3. Nếu không thấy default database, chạy lệnh SHOW DATABASES; trong query editor, sau đó nhấn Run.
  4. Nhấn vào biểu tượng làm mới và kiểm tra xem default đã xuất hiện chưa.

Tạo Bảng

Chúng ta sẽ tạo 4 bảng mới:

  • customer_csv
  • sales_csv
  • customer_parquet
  • sales_parquet

Tạo Bảng customer_csv

  1. Mở Query editor.

  2. Nhấn vào + để thêm tab mới.

  3. Sao chép và chạy lệnh tạo bảng.

    CREATE EXTERNAL TABLE customers_csv (
card_id bigint,
customer_id bigint,
lastname string,
firstname string,
email string,
address string,
birthday string,
country string)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
STORED AS INPUTFORMAT
'org.apache.hadoop.mapred.TextInputFormat'
OUTPUTFORMAT
'org.apache.hadoop.hive.ql.io.HiveIgnoreKeyTextOutputFormat'
LOCATION
's3://athena-workshop-<account id>/basics/csv/customers/'
TBLPROPERTIES (
'areColumnsQuoted'='false',
'classification'='csv',
'columnsOrdered'='true',
'compressionType'='none',
'delimiter'=',',
'skip.header.line.count'='1',
'typeOfData'='file');

  4. Nhấn Run.

  5. Kiểm tra trong Query Results để đảm bảo trạng thái Completed.

Tạo Bảng customer_parquet

  1. Mở Query editor, thêm tab mới.

  2. Sao chép và chạy lệnh tạo bảng.

    CREATE EXTERNAL TABLE `customers_parquet`(
card_id bigint,
customer_id bigint,
lastname string,
firstname string,
email string,
address string,
birthday string)
PARTITIONED BY (
`country` string)
ROW FORMAT SERDE
'org.apache.hadoop.hive.ql.io.parquet.serde.ParquetHiveSerDe'
STORED AS INPUTFORMAT
'org.apache.hadoop.hive.ql.io.parquet.MapredParquetInputFormat'
OUTPUTFORMAT
'org.apache.hadoop.hive.ql.io.parquet.MapredParquetOutputFormat'
LOCATION
's3://athena-workshop-<account id>/basics/parquet/customers/'
TBLPROPERTIES (
'classification'='parquet',
'compressionType'='none',
'partition_filtering.enabled'='true',
'typeOfData'='file');

MSCK REPAIR TABLE customers_parquet;

SHOW PARTITIONS customers_parquet;

  3. Nhấn Run.

  4. Chạy lệnh MSCK REPAIR TABLE customers_parquet; để cập nhật partition.

  5. Chạy SHOW PARTITIONS customers_parquet; để kiểm tra partition.

Phân Vùng Dữ Liệu Trong Athena

Bằng cách phân vùng dữ liệu, bạn có thể giảm lượng dữ liệu được quét khi chạy truy vấn, giúp cải thiện hiệu suất và giảm chi phí.

Athena hỗ trợ Apache Hive-style partitions, nơi đường dẫn chứa các cặp khóa-giá trị (ví dụ: country=us/ hoặc year=2021/month=01/).

Để nạp thêm partition vào bảng đã phân vùng, sử dụng lệnh sau:

MSCK REPAIR TABLE table_name;

So sánh Hiệu Suất Giữa Các Bảng

Bây giờ, sau khi đã tạo các bảng, hãy chạy các truy vấn để so sánh hiệu suất của chúng.

Các bước thực hiện:

  1. Quay lại Trình chỉnh sửa truy vấn (Query editor).

  2. Nhấp vào biểu tượng + để thêm tab mới.

  3. Sao chép và dán truy vấn dưới đây vào trình chỉnh sửa truy vấn:

    /* Lets find the top 10 products for Australia from the CSV table*/
select c.country, s.product_id, count(timestamp) as total_transactions
from customers_csv c join sales_csv s on c.customer_id = s.customer_id
where country ='Australia'
group by c.country,s.product_id
order by 1,3 desc
limit 10;

/* Lets find the top 10 products for Australia from the parquet tables*/
select c.country, s.product_id, count(timestamp) as total_transactions
from customers_parquet c join sales_parquet s on c.customer_id = s.customer_id
where country ='Australia'
group by c.country,s.product_id
order by 1,3 desc
limit 10;

/* Lets find the top 10 biggest spending customers from the sales CSV table */
select customer_id, sum(cast(price as decimal(6,2))) as total_sales
from sales_csv s 
group by customer_id
order by 2 desc
limit 10;

/* Lets find the top 10 biggest spending customers from the sales parquet table */
select customer_id, sum(cast(price as decimal(6,2))) as total_sales
from sales_parquet s 
group by customer_id
order by 2 desc
limit 10;

  4. Chạy các truy vấn từng cái một, bôi đen từng khối lệnh SQL và nhấn Run.

  5. Trong phần Kết quả truy vấn (Query Results), ghi lại các giá trị Thời gian chạy (Run Time) và Dữ liệu quét (Data Scanned).
